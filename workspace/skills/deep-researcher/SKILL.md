# База знаний: Агент-дипресёрчер (Яндекс Директ)
**Продукт:** SyncraLab AI Agents  
**Версия:** 1.0  
**Область:** Конкурентный анализ и рыночные исследования для performance-кампаний

---

## 1. РОЛЬ И ЗОНА ОТВЕТСТВЕННОСТИ

Агент-дипресёрчер — второе звено в пайплайне DISCOVERY. Получает `data-report.md` от аналитика и обогащает картину внешним контекстом: конкуренты, рынок, аудитория, тренды.

**Делает:**
- Анализирует конкурентов в Директе (объявления, офферы, позиционирование)
- Оценивает уровень конкуренции в аукционе
- Выявляет рыночные тренды и сезонность спроса
- Находит незакрытые потребности аудитории
- Передаёт `research-brief.md` агенту-гипотезатору

**Не делает:** не формулирует гипотезы, не меняет кампании, не пишет тексты объявлений.

## 2. ИСТОЧНИКИ ДАННЫХ И МЕТОДЫ

### 2.1 Яндекс Вордстат
**Что даёт:** реальный спрос по ключевым запросам, динамика, сезонность, регионы.

**Как использовать:**
- Проверять динамику спроса за 12 месяцев — выявляет сезонность и общий тренд рынка
- Сравнивать частотность целевых ключей с текущим охватом кампании
- Искать кластеры запросов, которые не покрыты текущими кампаниями
- Смотреть «похожие запросы» — находит смежные потребности аудитории

**Сигналы:**
- Спрос растёт → рынок расширяется, есть смысл масштабировать
- Спрос падает → конкуренция за сужающийся рынок, нужна дифференциация
- Острая сезонность → планировать бюджеты заранее, не в пик

### 2.2 Парсинг объявлений конкурентов
**Что даёт:** реальные тексты, офферы, УТП, призывы к действию конкурентов в Директе.

**Как анализировать объявления:**

1. **Заголовки** — что обещают, на какую боль давят
2. **Офферы** — скидки, гарантии, сроки, бесплатные опции
3. **Расширения** — быстрые ссылки, уточнения, цены (структура предложения)
4. **Посадочные страницы** — соответствие объявления и лендинга

**Матрица анализа конкурентов:**

| Конкурент | Главный оффер | УТП | CTA | Слабое место |
|-----------|--------------|-----|-----|-------------|
| {имя} | {оффер} | {УТП} | {призыв} | {что не закрывают} |

**Что искать:**
- Повторяющиеся паттерны у всех → это «гигиена рынка», без этого нельзя
- Что никто не говорит → потенциальное незанятое УТП
- Слабые офферы у лидеров → возможность атаки через сильный оффер

### 2.3 SimilarWeb / SEMrush / SpyFu
**Что даёт:** оценка трафика конкурентов, каналы, ключевые слова, рекламные расходы.

**Ключевые метрики для анализа:**
- Доля платного трафика у конкурентов (насколько они зависят от Директа)
- Топ-ключи конкурентов в платной выдаче
- Оценочный рекламный бюджет (позволяет понять уровень конкуренции)
- Bounce rate на посадочных конкурентов (косвенно — качество их лендингов)

**Важно:** данные этих инструментов — оценочные, не точные. Использовать как ориентир, не как факт.

### 2.4 Отраслевые отчёты и бенчмарки
**Что даёт:** средние показатели по рынку (CTR, CPA, конверсия) для сравнения с данными клиента.

**Источники для России:**
- Яндекс Рекламные технологии — официальные бенчмарки по отраслям
- Data Insight — e-commerce метрики
- Mediascope — аудиторные данные
- Отраслевые исследования VC.ru, Коммерсантъ, РБК

**Как применять:**
- Сравнивать CPA клиента со средним по отрасли
- Определять, является ли проблема клиента общерыночной или специфической
- Находить ниши с низкой конкуренцией (низкий средний CPC по отрасли)

### 2.5 Аудиторные сигналы (соцсети, форумы, отзывы)
**Что даёт:** язык аудитории, реальные боли, возражения, критерии выбора.

**Где искать:**
- Отзывы на Яндекс Картах, Отзовике, 2ГИС — реальные слова клиентов
- Тематические форумы и Telegram-каналы — незафильтрованные мнения
- Комментарии под рекламой конкурентов в VK — возражения и вопросы
- Вопросы на Яндекс Кью — чего люди не понимают в продукте/нише

**Что фиксировать:**
- Формулировки боли (дословно — пойдут в тексты объявлений копирайтеру)
- Частые возражения (цена, доверие, сроки, сложность)
- Критерии выбора (по чему выбирают между конкурентами)
- Эмоциональные триггеры (страх, срочность, статус, экономия)

## 3. АНАЛИЗ КОНКУРЕНТНОЙ СРЕДЫ

### 3.1 Типы конкурентов
| Тип | Описание | Приоритет анализа |
|-----|---------|------------------|
| Прямые | Тот же продукт, та же аудитория | Высокий |
| Косвенные | Другой продукт, та же потребность | Средний |
| Заменители | Альтернативное решение проблемы | Средний |
| Информационные | Статьи, агрегаторы, сравнения | Низкий |

### 3.2 Оценка конкурентности аукциона
**Сигналы высокой конкуренции:**
- Средний CPC выше отраслевого бенчмарка в 1.5+ раза
- 5+ рекламодателей на первой странице по целевым запросам
- Короткие циклы обновления объявлений у конкурентов (активно тестируют)
- Агрессивные скидочные офферы (ценовая война)

**Что делать при высокой конкуренции:**
- Искать низкочастотные, но конверсионные запросы (хвост семантики)
- Дифференцироваться по офферу, а не конкурировать ставками
- Тестировать нестандартное время показа (когда конкуренты снижают ставки)
- Работать с аудиторными таргетингами, а не только ключами

### 3.3 Карта конкурентного позиционирования
Строится по двум осям (выбираются под нишу), например:
- Ось X: Цена (дёшево → дорого)
- Ось Y: Скорость/сервис (медленно → быстро)

Цель: найти незанятый квадрант — возможность для дифференциации клиента.

## 4. АНАЛИЗ ТРЕНДОВ И СЕЗОННОСТИ

### 4.1 Типы сезонности
| Тип | Пример | Действие |
|-----|--------|---------|
| Предсказуемая годовая | Новый год, 8 марта, 1 сентября | Готовить кампании за 3–4 недели |
| Погодная | Кондиционеры летом, шины осенью | Автоправила по температуре/осадкам |
| Событийная | Чемпионаты, выборы, кризисы | Мониторинг новостного фона |
| Недельная | B2B: пики пн-вт, спад пт-вс | Корректировки ставок по дням |

### 4.2 Как читать тренды Вордстата
- Рост запроса >20% YoY → растущий рынок, инвестировать сейчас
- Падение запроса >20% YoY → угасающий спрос, искать смежные ниши
- Пикообразный график → сезонный продукт, планировать бюджет под пики
- Плоский график → стабильный спрос, фокус на доле рынка

## 5. СТРУКТУРА ВЫХОДНОГО ДОКУМЕНТА

Дипресёрчер передаёт `research-brief.md` гипотезатору. Документ должен содержать:

### Обязательные разделы:

**1. Конкурентная карта**
- Список 3–7 ключевых конкурентов с их основными офферами
- Что рынок говорит «в один голос» (обязательный минимум)
- Незанятые позиции в позиционировании

**2. Анализ спроса**
- Динамика ключевых запросов (растёт/падает/стабильно)
- Сезонность (есть/нет, когда пики)
- Незакрытые запросы (есть спрос, нет рекламодателей)

**3. Аудиторные инсайты**
- Топ-3 боли аудитории (дословные формулировки)
- Топ-3 возражения
- Критерии выбора поставщика/продукта

**4. Зоны возможностей**
- Где конкуренты слабы (плохие офферы, слабые посадочные)
- Где спрос не удовлетворён
- Какие форматы/каналы конкуренты не используют

**5. Данные для гипотезатора**
- Список наблюдений в формате «факт → потенциал»
- Приоритизация: высокий / средний / низкий потенциал

## 6. ТИПИЧНЫЕ ОШИБКИ ПРИ РЕСЁРЧЕ

| Ошибка | Последствие | Как избежать |
|--------|------------|-------------|
| Анализировать только прямых конкурентов | Упустить косвенных, которые перехватывают спрос | Проверять смежные категории запросов |
| Доверять данным SimilarWeb как точным | Неверные выводы о бюджетах конкурентов | Использовать как ориентир, перекрёстно проверять |
| Игнорировать региональную специфику | Нерелевантные бенчмарки | Всегда фильтровать данные по гео клиента |
| Копировать офферы конкурентов | Слияние с рынком, нет дифференциации | Искать то, чего конкуренты НЕ говорят |
| Ресёрч без привязки к данным аналитика | Выводы не решают реальную проблему | Начинать с `data-report.md`, не с нуля |

---

## 7. ПОРОГОВЫЕ ЗНАЧЕНИЯ ДЛЯ ВЫВОДОВ

| Наблюдение | Порог для фиксации |
|-----------|-------------------|
| Конкурент использует оффер | 3+ конкурента → это «норма рынка» |
| Запрос незакрыт рекламой | < 3 рекламодателей на странице |
| Спрос растёт | > 15% рост WoW или > 30% YoY |
| Спрос падает | > 15% падение WoW или > 25% YoY |
| Высокая конкуренция | > 5 рекламодателей + CPC выше бенчмарка |
