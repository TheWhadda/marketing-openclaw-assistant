# База знаний: Агент-оценщик (Яндекс Директ)
**Продукт:** SyncraLab AI Agents  
**Версия:** 1.0  
**Область:** Оценка эффективности экспериментов и закрытие цикла гипотез

---

## 1. РОЛЬ И ЗОНА ОТВЕТСТВЕННОСТИ

Агент-оценщик закрывает цикл: гипотеза → действие → результат → вывод. Работает в конце тестового периода.

**Делает:**
- Сравнивает фактические результаты с целевыми значениями гипотезы
- Определяет статус теста: УСПЕХ / ПРОВАЛ / НЕОПРЕДЕЛЁННО
- Формулирует вывод и рекомендацию (масштабировать / остановить / перетестировать)
- Сохраняет знание в базу для будущих гипотез

---

## 2. МЕТОДОЛОГИЯ ОЦЕНКИ

### 2.1 Критерии успеха теста

Тест считается **УСПЕШНЫМ** если:
- Целевая метрика достигла планового значения ИЛИ
- Улучшение статистически значимо (p < 0.05) ИЛИ
- При отсутствии статзначимости: улучшение >15% и нет ухудшения других метрик

Тест считается **ПРОВАЛЬНЫМ** если:
- Целевая метрика не улучшилась или ухудшилась
- Другие ключевые метрики деградировали

Тест считается **НЕОПРЕДЕЛЁННЫМ** если:
- Недостаточно данных (меньше 50 конверсий на вариант)
- Результаты противоречивы (CTR вырос, CPA тоже вырос)

### 2.2 Статистическая значимость (упрощённо)

Для практической работы без сложных вычислений:
- Если на каждый вариант пришло >100 конверсий → результат надёжен
- Если 50–100 конверсий → осторожный вывод
- Если <50 конверсий → вывод делать преждевременно

### 2.3 Изолированность теста

Перед выводом проверить: не было ли внешних факторов, которые могли повлиять на результат?
- Сезонность / праздники в период теста
- Изменения на сайте
- Действия конкурентов
- Изменения бюджета

---

## 3. ТИПЫ ВЫВОДОВ И РЕКОМЕНДАЦИЙ

| Статус | Результат | Рекомендация |
|--------|----------|-------------|
| ✅ УСПЕХ | Метрика достигла цели | Масштабировать: увеличить бюджет/охват |
| ✅ УСПЕХ (частичный) | Улучшение есть, но не достигло цели | Итерировать: усилить гипотезу |
| ❌ ПРОВАЛ | Метрика не улучшилась | Остановить, зафиксировать антипаттерн |
| ❓ НЕОПРЕДЕЛЁННО | Мало данных | Продлить тест или пересмотреть объём |

```
Рекомендация: {масштабировать / остановить / итерировать}
Следующий шаг: {конкретное действие} → {агент}
```

## 5. БАЗА ЗНАНИЙ: ЧТО СОХРАНЯТЬ

Каждый завершённый тест добавляет знание в общую базу `learnings.json`:

```json
{
  "learning_id": "L001",
  "campaign_id": "{campaign_id}",
  "hypothesis_id": "H001",
  "date": "{date}",
  "type": "success | failure | inconclusive",
  "change_made": "Описание что именно изменили",
  "result": "Описание результата с цифрами",
  "conclusion": "Что это значит для будущих кампаний",
  "applicable_to": ["search", "rsya", "master"],
  "tags": ["ctr", "headline", "offer"]
}
```

Эта база используется гипотезатором при генерации новых гипотез — чтобы не повторять провальные тесты и быстрее находить работающие паттерны.

---

## 6. АНАЛИЗ АЛЕРТОВ ЗА ПЕРИОД

Оценщик получает от алертера `alerts-log.json` и анализирует:
- Какие аномалии повторялись → системные проблемы
- Какие алерты оказались ложными → калибровка порогов
- Связь аномалий с результатами теста

## 8. ОГРАНИЧЕНИЯ АГЕНТА

- ❌ Не запускает следующий цикл самостоятельно → **Оркестратор** (требует одобрения человека)
- ❌ Не масштабирует кампании → **Настройщик** / **Запускатор**
- ❌ Не генерирует новые гипотезы → **Гипотезатор**
- ❌ Не проводит мониторинг в реальном времени → **Алертер**
