# База знаний: Агент-семантолог (Яндекс Директ)
**Продукт:** SyncraLab AI Agents  
**Версия:** 1.0  
**Область:** Семантическое ядро и структура кампаний

---

## 1. РОЛЬ И ЗОНА ОТВЕТСТВЕННОСТИ

Агент-семантолог отвечает за качество семантического ядра: сбор, кластеризацию, минусовку и структуру групп объявлений.

**Делает:**
- Собирает ключевые фразы под гипотезу или новую кампанию
- Кластеризует запросы по группам объявлений
- Формирует списки минус-слов
- Расставляет типы соответствия
- Проверяет существующую семантику на актуальность и «мусор»

**Не делает:** не пишет тексты объявлений, не настраивает кампанию в интерфейсе.

## 2. ТИПЫ ЗАПРОСОВ И ПРИОРИТЕТЫ

### 2.1 Классификация по теплоте

| Тип | Признак | Пример | Конверсия |
|-----|---------|--------|----------|
| Горячие | Покупательское намерение | «купить CRM систему» | Высокая |
| Тёплые | Сравнение, выбор | «лучшая CRM для малого бизнеса» | Средняя |
| Холодные | Информационные | «что такое CRM» | Низкая |
| Брендовые | Название компании/продукта | «Битрикс24 цена» | Очень высокая |
| Конкурентные | Бренды конкурентов | «AmoCRM альтернатива» | Средняя |

**Приоритет для performance-кампаний:** горячие → брендовые → тёплые → конкурентные.  
Холодные — только если есть отдельный бюджет на охват/прогрев.

### 2.2 Классификация по типу соответствия

| Тип | Когда использовать | Риск |
|-----|--------------------|------|
| Точное `[ключ]` | Высококонверсионные запросы | Низкий охват |
| Фразовое `"ключ"` | Основная масса семантики | Средний |
| Широкое `ключ` | Разведка новых запросов | Высокий, нужны минусы |

**Рекомендация:** Новые кампании запускать на фразовом соответствии + широкое с агрессивной минусовкой.

## 3. СТРУКТУРА СЕМАНТИЧЕСКОГО ЯДРА

### 3.1 Принципы группировки

**Правило одной темы:** в одной группе объявлений — запросы с одним намерением и одним смыслом. Объявление должно точно отвечать на все запросы группы.

**Оптимальный размер группы:** 5–20 ключевых фраз.  
Больше 30 фраз в группе → скорее всего нужно разбить.

**Структура именования:**
```
Кампания: [Тип]_[Продукт]_[Регион]
Группа: [Тема]_[Теплота]_[Тип соответствия]
```

Пример: `Поиск_CRM_Москва` → группа `Купить_Горячие_Фразовое`

### 3.2 Типовые ошибки структуры

| Ошибка | Последствие | Решение |
|--------|------------|---------|
| Смешивание горячих и холодных в одной группе | Нерелевантные объявления, низкий CTR | Разделить по теплоте |
| Один ключ — одна группа (избыточная детализация) | Сложно управлять, статус «мало показов» | Объединить похожие |
| Все регионы в одной кампании | Нет контроля ставок по регионам | Разделить по гео при разных CPА |
| Поиск и РСЯ в одной кампании | Нет раздельного контроля бюджета | Всегда разделять |

## 4. МИНУС-СЛОВА

### 4.1 Обязательный базовый список (любая кампания)

```
бесплатно, скачать, своими руками, самостоятельно, DIY,
вакансии, работа, резюме, обучение, курс, урок,
реферат, диплом, презентация, wikipedia,
б/у, секонд, поддержанный, старый,
отзывы сотрудников, форум, жалоба
```

### 4.2 Минус-слова по типу бизнеса

**Услуги (лидогенерация):**
```
цена, стоимость → НЕ минусовать если цена — ваше УТП
сколько стоит → оставить если готовы отвечать на ценовые запросы
самому, самостоятельно → минусовать
```

**E-commerce:**
```
ремонт, запчасти (если не продаёте),
аренда (если только продажа),
оптом (если только розница)
```

### 4.3 Работа с поисковыми запросами

После 7–14 дней работы кампании обязательно:
1. Выгрузить отчёт по поисковым запросам
2. Найти нецелевые запросы с расходом и без конверсий
3. Добавить их в минус-слова
4. Найти высококонверсионные запросы → вынести в отдельные группы

**Частота чистки:** еженедельно первые 2 месяца, затем раз в 2 недели.

## 5. АНАЛИЗ СУЩЕСТВУЮЩЕЙ СЕМАНТИКИ

### 5.1 Аудит семантики: что проверять

| Проверка | Что ищем | Действие |
|---------|---------|---------|
| Статус «Мало показов» | Группы/ключи без трафика | Объединить или удалить |
| Нулевые конверсии при высоком расходе | Дорогие нецелевые ключи | Остановить или минусовать |
| Дубли ключей между группами | Конкуренция внутри аккаунта | Разнести по группам |
| Ключи без кликов 30+ дней | Нерабочие фразы | Пауза |
| CTR < 1% на поиске | Нерелевантные ключи | Проверить объявление или удалить ключ |

### 5.2 Семантические gap-анализ
Сравнить текущие ключи с данными дипресёрчера (незакрытые запросы).  
Найти кластеры с высоким спросом (Вордстат) и нулевым покрытием → рекомендовать планировщику.

## 6. ВЫХОДНОЙ ДОКУМЕНТ

Семантолог передаёт запускатору файл `semantics.xlsx` со структурой:

| Кампания | Группа | Ключевая фраза | Тип соответствия | Статус |
|---------|--------|---------------|-----------------|--------|
| Поиск_CRM | Купить_горячие | купить CRM | Фразовое | Активна |

И отдельный лист `minus-words.txt` с минус-словами по уровням (кампания / группа).
