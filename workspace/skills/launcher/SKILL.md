# База знаний: Агент-запускатор (Яндекс Директ)
**Продукт:** SyncraLab AI Agents  
**Версия:** 1.0  
**Область:** Технический запуск рекламных кампаний в Яндекс Директ

---

## 1. РОЛЬ И ЗОНА ОТВЕТСТВЕННОСТИ

Агент-запускатор — технический специалист финальной стадии перед эфиром. Получает все готовые материалы (семантика, тексты, баннеры, настройки) и производит финальный запуск кампании или теста.

**Делает:**
- Проверяет комплектность всех материалов по чеклисту
- Загружает кампанию / вносит изменения в Директ
- Устанавливает лимиты бюджета и расписание
- Фиксирует момент запуска с точными параметрами
- Передаёт контролёру данные для проверки

**Не делает:** не создаёт материалы, не принимает стратегических решений, не оценивает результаты.

---

## 2. ПРЕДЗАПУСКНЫЙ ЧЕКЛИСТ

### 2.1 Материалы (всё должно быть готово)

**Семантика:**
- [ ] Список ключевых фраз загружен и проверен семантологом
- [ ] Минус-слова добавлены на уровне кампании и групп
- [ ] Типы соответствия расставлены корректно
- [ ] Нет дублей между группами

**Тексты объявлений:**
- [ ] Заголовок 1: не более 56 символов
- [ ] Заголовок 2: не более 30 символов
- [ ] Текст объявления: не более 81 символа
- [ ] Быстрые ссылки: 4 штуки, каждая ≤ 30 символов
- [ ] Уточнения: 4+ штуки
- [ ] Отображаемый URL: заполнен, соответствует странице
- [ ] UTM-метки: проставлены на всех ссылках

**Баннеры (для РСЯ / МК / Смарт):**
- [ ] Все обязательные форматы подготовлены (см. раздел 4)
- [ ] Баннеры прошли проверку на соответствие требованиям Яндекса
- [ ] Нет текста мелче 14px
- [ ] Логотип читаем

**Настройки кампании:**
- [ ] Стратегия показов выбрана и согласована с планировщиком
- [ ] Дневной бюджет установлен
- [ ] Временной таргетинг настроен
- [ ] Геотаргетинг: регионы выбраны корректно
- [ ] Счётчик Метрики привязан
- [ ] Цели конверсий подключены и проверены
- [ ] Ключевые цели выставлены

### 2.2 Технические проверки

- [ ] Посадочная страница открывается и загружается < 3 сек
- [ ] Форма на посадочной работает (тестовая заявка)
- [ ] UTM-метки корректно передаются в Метрику
- [ ] Цели срабатывают при тестовом действии
- [ ] Номер телефона / email в объявлении актуален

## 3. ПОРЯДОК ЗАПУСКА

### 3.1 Новая кампания
```
1. Создать структуру кампании в Директе (кампания → группы → объявления)
2. Загрузить семантику в группы
3. Загрузить тексты объявлений
4. Загрузить баннеры (если есть)
5. Настроить параметры кампании (стратегия, бюджет, гео, время)
6. Привязать счётчик Метрики и цели
7. Проставить UTM-метки
8. Отправить на модерацию
9. Дождаться прохождения модерации
10. Установить дневной лимит = плановый бюджет / дни теста
11. Нажать «Запустить»
12. Зафиксировать время старта в launch-log.json
13. Уведомить контролёра
```

### 3.2 Изменения в существующей кампании
```
1. Создать копию текущей группы/кампании (бэкап)
2. Внести изменения
3. Проверить изменения по чеклисту
4. Применить
5. Зафиксировать что именно изменено и когда
6. Уведомить контролёра
```

### 3.3 A/B тест
```
1. Создать экспериментальную группу объявлений (не трогать контрольную)
2. Распределить трафик: 50/50 или 70/30 (контроль/эксперимент)
3. Убедиться что обе группы показываются одновременно
4. Зафиксировать базовые метрики контрольной группы на момент старта
5. Установить дату окончания теста
6. Уведомить аналитика и оценщика о старте
```

## 4. ФОРМАТЫ БАННЕРОВ ДЛЯ РСЯ И МАСТЕР КАМПАНИЙ

### 4.1 Обязательные форматы РСЯ
| Формат | Размер | Вес |
|--------|--------|-----|
| Горизонтальный | 1080×607 px | до 150 КБ |
| Квадратный | 1080×1080 px | до 150 КБ |
| Вертикальный | 607×1080 px | до 150 КБ |

### 4.2 Мастер кампаний
- Изображение: минимум 1:1 (квадрат) и 16:9 (горизонталь)
- Логотип: квадратный, минимум 200×200 px
- Видео (опционально): до 60 сек, MP4

### 4.3 Смарт-баннеры
- Генерируются автоматически из фида
- Запускатор проверяет качество фида перед запуском:
  - [ ] Все обязательные поля заполнены (name, price, url, picture)
  - [ ] Цены актуальны
  - [ ] Ссылки ведут на существующие страницы
  - [ ] Нет товаров с нулевым стоком (если настроен фильтр)

## 5. UTM-МЕТКИ: СТАНДАРТ SYNCRALAB

Единый формат UTM для всех кампаний:

```
utm_source=yandex
utm_medium=cpc
utm_campaign={campaign_name}_{campaign_type}
utm_content={ad_group_name}
utm_term={keyword}
```

**Обязательные параметры Директа (динамические):**
```
utm_campaign={campaign_id}
utm_content={ad_id}
utm_term={keyword}
```

Все ссылки проверяются через валидатор UTM перед запуском.

## 6. РАБОТА СО СТРАТЕГИЯМИ СТАВОК

### 6.1 Когда использовать какую стратегию

| Ситуация | Стратегия |
|---------|-----------|
| Новая кампания, нет данных | Ручное управление ставками |
| Есть >10 конверсий/нед, CPA нужно снизить | Оптимизация конверсий (целевой CPA) |
| E-commerce, нужен ROAS | Оптимизация рентабельности (целевой ROAS) |
| Нужен максимальный охват | Максимум кликов |
| Мастер кампаний | Автоматически (менять не нужно) |

### 6.2 Период обучения стратегий
- Ручное управление: нет периода обучения
- Оптимизация конверсий: **2–4 недели**, нужно минимум 10 конверсий/нед
- Оптимизация ROAS: **3–4 недели**, нужно минимум 20 конверсий/нед

**Критично:** в период обучения автостратегии не вносить изменений в кампанию — это сбрасывает обучение.

## 7. МОДЕРАЦИЯ: ТИПИЧНЫЕ ПРИЧИНЫ ОТКЛОНЕНИЯ

| Причина | Решение |
|---------|---------|
| Превосходные степени («лучший», «№1») | Убрать или добавить подтверждение |
| Контактные данные в тексте | Убрать телефон/email из текста объявления |
| Несоответствие объявления посадочной | Синхронизировать оффер |
| Запрещённая тематика (медицина, финансы) | Добавить дисклеймеры, загрузить документы |
| Кириллица в URL | Использовать латиницу в отображаемом URL |

Срок модерации: обычно 1–3 часа, до 24 часов в сложных тематиках.
