# База знаний: Агент-настройщик (Яндекс Директ)
**Продукт:** SyncraLab AI Agents  
**Версия:** 1.0  
**Область:** Настройка таргетингов и параметров кампаний

---

## 1. РОЛЬ И ЗОНА ОТВЕТСТВЕННОСТИ

Агент-настройщик переводит стратегию и материалы в конкретные настройки рекламного кабинета Яндекс Директ. Педантичен и методичен — каждая настройка документируется.

---

## 2. ПАРАМЕТРЫ КАМПАНИИ: ЧЕКЛИСТ НАСТРОЙКИ

### 2.1 Базовые параметры
- [ ] Название кампании по стандарту: `[Тип]_[Продукт]_[Регион]_[Дата]`
- [ ] Стратегия показов: выбрана согласно рекомендации планировщика
- [ ] Дневной бюджет: установлен лимит
- [ ] Расписание показов: настроено (или 24/7 с корректировками)
- [ ] Счётчик Метрики: привязан
- [ ] Ключевые цели: установлены с ценностью конверсии

### 2.2 Геотаргетинг
- [ ] Регионы показа: выбраны согласно плану
- [ ] Расширенный геотаргетинг: вкл/выкл осознанно
  - Вкл: если клиент ищет услугу в другом городе (переехать, командировка)
  - Выкл: если нужны только жители региона

### 2.3 Временной таргетинг
| Бизнес-тип | Рекомендация |
|-----------|-------------|
| B2B услуги | Пн–Пт 9:00–19:00, понижение -50% на выходных |
| B2C срочные услуги | 24/7 |
| E-commerce | 24/7 с повышением в пиковые часы (19:00–22:00) |
| Локальный бизнес | Часы работы + 1 час до/после |

### 2.4 Корректировки ставок

**По устройствам:**
- Мобильные CR < 50% от десктопа → корректировка -30% на мобильные
- Мобильные CR сопоставим → без корректировки
- Никогда не ставить -100% на мобильные без анализа (теряем трафик)

**По аудитории:**
- Ретаргетинг (были на сайте, не купили) → +30–50%
- Похожие аудитории (look-alike) → +15–25%
- Аудитория конкурентов (если собрана) → +20–30%

**По полу/возрасту:**
- Только на основе данных аналитика. Без данных — не трогать.

## 3. НАСТРОЙКА СТРАТЕГИЙ СТАВОК

### 3.1 Ручное управление ставками
- Использовать при запуске новых кампаний
- Ставка = CPA_target × CR_ожидаемый
- Пример: CPA_target 1500₽, CR 3% → ставка = 1500 × 0.03 = 45₽

### 3.2 Оптимизация конверсий (авто)
- Включать только при наличии >10 конверсий в неделю
- Установить недельный бюджет (не дневной) для лучшей оптимизации
- Не трогать 2–4 недели после включения

### 3.3 Оптимизация рентабельности
- Включать при >20 конверсий в неделю
- Установить целевой ROAS на 10–15% выше желаемого (алгоритм занижает)

## 4. НАСТРОЙКА ЦЕЛЕЙ В МЕТРИКЕ

Приоритет целей (сверху вниз — что считать конверсией):
1. Отправка формы / заявки (микроконверсия → макроконверсия)
2. Звонок (через коллтрекинг)
3. Нажатие на кнопку CTA
4. Визит на страницу «Спасибо»
5. Время на сайте > N минут (только как вспомогательная)

**Важно:** конверсионная цель должна отражать реальное бизнес-действие, а не просто посещение страницы.
